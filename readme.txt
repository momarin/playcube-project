# Projeto PlayCube para teste de vaga Front-End;

# Aterações principais em relação ao design original:
    ● O design principal tem seus espaçamentos, em sua maioria, em divisores de 5. Como o Bootstrap usa o Rem e os divisores de 4, tomei a liberdade para deixar alguns espaçamentos com um valor aproximado ao original, usando as classes do Bootstrap.

# Dificuldades encontradas:
    - QUANTO AO HTML
        ● Compreendo que usei demais a classe "container", quando ela deve ser usada somente no container principal (de preferência). Entretanto, quando percebi meu erro, o HTML e CSS já estavam prontos. Lembro que fui adicionando a classe porque ela resolvia o problema de margin e padding das rows no desktop, mas quando tentei remove-las e consertar, já era tarde demais e, acredito, teria que refazer o html (farei isso no futuro);
        ● Ainda não aprendi a aplicar corretamente as arias de acessibilidade e a lidar com o html semântico. Algumas partes rerão as arias de forma correta e, em outras, não. Isso também será corrigido no futuro.

    - QUANTO AO CSS
        ● Quanto ao footer, não consegui dividir a lista de links tal qual está na versão MD. Depois de várias tentatuivas frustradas, deixei o MD como no SM. Até consegui fazer em separado com propriedades de CSS, me basendo no site do New York Times, mas ao tentar aplicar no meu html, o desktop não funcionava. Acredito que alguma classe estava conflitando com a outra, então deixei essa parte para um conserto futuro;

        ● Não usei os fades com os .png disponibilizados, preferi fazer pelo próprio CSS, pois consegui controlar mais facilmente;

        ● Fiz mais de um arquivo de CSS por uma questão de praticidade. Não sei se é a melhor prática ou se apliquei corretamente as classes em seu devido arquivo, mas para mim facilitou muito na hora de alterar o responsivo, por exemplo;

    - QUANTO AO JAVASCRIPT
        ● Esse é o meu maior fraco e ainda sofro muito para "falar" JS, mas, ao olhar para um design, eu sei exatamente o que vou precisar, então eu acabo pegando fragmentos de código e vou montando, pesquisando e procurando na internet as soluções mais adequadas;
        
        ● A página está consumindo a api através de ID, então a análise pode ser feita através de qualquer filme. Uma observação importante é que filmes com títulos grandes ou com muita gente envolvida nas sessões "dirigido por" ou "escrito por" alterarão consideravelmente o design.

# Aprendizado:
    ● Esse foi um grande desafio para mim, o maior projeto que eu já tinha feito para testar minhas habilidades. O que fica de lição principal para é organizar minhas ideias e depois passá-las ao código;
    ● Apesar de eu ter sentido dificuldade em lidar com o Bootstrap no início, ao final do desafio eu já estava achando bem melhor usá-lo do que ficar só com o CSS vanilla;
    ● O HTML me assustou pelo tamanho, mas quando iniciei o JS, o primeiro foi encolhendo e fazendo mais sentido para mim.

# Observações gerais:
    Alguns comentários estão em inglês e outros em português. Acredito que isso acontecia de acordo com meu nível de cansaço e só percebi na revisão final. Deixei os divisores de seção em inglês, pois foi assim que comecei, mas alterei os comentários que notei para português, porém alguns podem ter passado desapercebidos.
